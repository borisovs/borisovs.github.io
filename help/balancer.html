The Balancer uses postgresql database server to store the data. Be sure you have installed and configured one.
Download and run perfomon_balancer.msi
Type in ip address and port which the balancer should listen
Type in DB address or hostname, DB port, DB user and password, also setup a connection pool
Choose the destination folder, run next and install
All these parameters you can set up by the Orca
Download the Orca and open perfomon_balancer.msi in it.
Also you can change any parameter after installation in the DESTINATION_FOLDER\conf\balancer.conf.xml
If you changed a configuration file, you need to restart the Balancer service in the services.msc
The Balancer uses postgresql database server to store the data. Be sure you have installed and configured one.
Ubuntu. Download and install perfomon_balancer_1.0.0_amd64.deb
sudo dpkg -i perfomon_balancer_1.0.0_amd64.deb
sudo apt-get install -f
RedHat. Download and install balancer_1.0.0_amd64.rpm
su - root
dnf localinstall balancer_1.0.0_amd64.rpm
Configuration
After installation, you should configure the balancer by changing parameters in the /etc/balancer.conf.xml
and start the balancer service
systemctl start balancer
Also you should enable starting the balancer during the booting process
systemctl enable balancer
Parameter Description
listen\address This is an address which a balancer will listen
listen\port This is a port which a balancer will open
logfile\path Balancer log file
database\address Hostname or IP address of a PostgreSQL database server
database\port Port which a PostgreSQL server is listening
database\pool The Connections count to the database server
database\dbname Name of the balancer database
database\user Database user
database\pass Password of the database user
If you changed configuration file, you need to restart the Balancer service
systemctl restart balancer
