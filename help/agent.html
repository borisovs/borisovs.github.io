Agents guide
  • Windows
  • Linux
Download and run perfomon_agent.msi
Configuration
After installation, you should configure the agent by changing parameters in the C:\Program Files\conf\agent.conf.xml
Parameter Description
endpoint\address Host where balancer is working
endpoint\port Port that balancer is listening
If you changed a configuration file, you need to restart the agent service in the services.msc
Orca

All these parameters you can set up by the Orca.

Download Orca and open perfomon_agent.msi then change values of the parameters (ENDPOINT_HOST, ENDPOINT_PORT) and run perfomon_agent.msi

The right way to deploy the agent in Active Directory is using the group policy this is a reason why perfomon_agent.msi doesn't ask you anything during the installation process.

  • Configure perfomon_agent.msi in Orca1
  • Create the new deploy policy at the domain controller2
  • Deploy the agent through the group policy3
Ubuntu. Download and install perfomon_agent_1.0.0_amd64.deb
sudo dpkg -i perfomon_agent_1.0.0_amd64.deb
sudo apt-get install -f
RedHat. Download and install perfomon_agent_1.0.0_amd64.rpm
su - root
dnf localinstall perfomon_agent_1.0.0_amd64.rpm
Configuration
After installation, you should configure the agent by changing parameters in the /etc/agent.conf.xml
and start the agent service
systemctl start agent
Also you should enable starting the agent service during the booting process
systemctl enable agent
Parameter Description
endpoint\address The host where a balancer is working
endpoint\port The port which a balancer is listening
logfile\path Agent log file
If you changed a configuration file, you need to restart the agent service
systemctl restart agent
